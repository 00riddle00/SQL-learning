
| Style Item  | Explanation                                                                            |
| ----------- | -------------------------------------------------------------------------------------- |
| Indentation | Style of how query blocks are visually nested using spaces and line breaks.            |
| Casing      | Style of uppercase/lowercase usage for SQL keywords, tables, and aliases.              |
| Alignment   | Style of vertically lining up SELECT columns, JOINs, and ON clauses for clarity.       |
| Ordering    | Style of arranging columns logically (IDs first, then names, then numeric values).     |
| Comments    | Style of documenting tasks and queries using consistent and readable comment blocks.   |
| Aliases     | Style of naming and using table or column aliases in a clear and systematic way.       |
| Joins       | Style of writing JOIN clauses, including structure, placement, and readability.        |
| Filtering   | Style of expressing WHERE conditions (IN, BETWEEN, comparison direction).              |
| Sorting     | Style of specifying ORDER BY rules, including explicit ASC/DESC usage.                 |
| Structure   | Style of organizing the whole file: task header → query → spacing.                     |
| Columns     | Style of choosing which columns to return and avoiding unnecessary `SELECT *`.         |
| Consistency | Style of repeating patterns (naming, formatting, aliasing) identically across tasks.   |
| Functions   | Style of using helper functions (DATE, LEFT, COUNT) in a predictable, clean manner.    |
| Formatting  | Style controlling whitespace, blank lines, semicolons, and general aesthetics.         |
| Semantics   | Style of choosing queries that precisely match task requirements without extra fields. |
| Readability | Style ensuring that anyone reading the query can understand its intent at a glance.    |
